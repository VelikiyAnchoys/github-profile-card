# GitHub Profile Card

Веб-приложение, в которое вводишь GitHub-юзернейм и получаешь карточку с информацией о пользователе: аватар, имя, био, статистика, последние репозитории и топ языков. Утилита для быстрого просмотра GitHub-профилей.

## Стек

HTML— семантическая разметка
CSS— кастомные свойства, Flexbox, анимации, адаптивная вёрстка
JavaScript — async/await, Fetch API, работа с DOM, localStorage
GitHub REST API v3 — публичное API без авторизации

Без фреймворков и библиотек: чистый ванильный стек.

## Как запустить локально

```
git clone <ссылка на репо>
cd test-cline
python -m http.server 8080
```

Затем открой в браузере: `http://localhost:8080`

## Деплой

Можно задеплоить на GitHub Pages:
1. Создать репозиторий на GitHub
2. Загрузить туда файлы `index.html`, `styles.css`, `script.js`
3. В настройках репозитория включить GitHub Pages (ветка `main`, корень `/`)

## Что работает

Поиск пользователя— ввод юзернейма + кнопка «Найти» или Enter
Карточка профиля — аватар, имя, био, количество репозиториев, подписчиков и подписок
Последние 5 репозиториев— с названием, описанием, языком, звёздами и форками
Сортировка репозиториев— по обновлению, по звёздам, по дате создания
Топ языков— подсчёт и визуализация языков программирования с цветными барами
Тёмная и светлая темы — переключатель с сохранением выбора в localStorage
Случайный пользователь — кнопка в виде кубика выбирает случайного известного разработчика
Недавние поиски — последние 5 юзернеймов сохраняются в localStorage, доступны по клику
Обработка ошибок — понятное сообщение при 404 (пользователь не найден) и других ошибках
Индикатор загрузки — спиннер с текстом «Загружаю...» во время запроса


## Что я узнала во время работы

Работа с GitHub REST API — научилась делать запросы к публичным эндпоинтам `/users/USERNAME` и `/users/USERNAME/repos`, обрабатывать ответы и ошибки (особенно 404).
CSS-переменные для тем— освоила технику переключения между тёмной и светлой темой через `data-theme` атрибут на `<html>` и набор CSS-переменных (`:root` и `[data-theme="light"]`). Тема сохраняется в `localStorage`.
Асинхронные запрос— закрепила работу с `async/await` и `fetch`, обработку ошибок через `try/catch`, параллельные запросы к разным эндпоинтам API.
localStorage для UI-состояния— сохраняла не только тему, но и историю поисков, что дало удобный UX без бэкенда.
